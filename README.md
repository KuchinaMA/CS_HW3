# p2p приложение с обходом NAT

Файлы в репозитории:
- `client.py` — программа для клиентов для всех 3-х случаев;
- `server.py` — вспомогательная для рандеву сервера;
- `traffic.pcapng` — запись сетевого трафика.

### Как работает программа

1. Клиенты регистрируются на сервере, узнают свои публичные адреса через NAT.
2. Сервер знакомит клиентов, передавая им адреса друг друга.
3. Оба клиента начинают слать UDP пакеты друг другу.
4. Когда пакет выходит наружу, NAT разрешает ответный трафик.
5. После "пробивания" клиенты общаются напрямую без сервера.

Сервер нужен только для знакомства, дальше общение идет напрямую между клиентами через NAT.

### Что видно в трафике?
Сначала запросы и ответы идут через сервер (192.168.100.10), а затем клиенты (192.168.100.3 и 192.168.100.4) общаются сежду собой напрямую. При более продробном рассмотрении записи можно увидеть, что это действительно сообщения и содержание сообщений.
